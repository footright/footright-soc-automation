# Demo Talk Track

## Goal

Use the interactive Footright demo to show how Footright LLC helps clients reduce alert noise, improve triage consistency, and accelerate security response.

## Opening

What I want to show you is not just another dashboard. This is how we help security teams automate the first layer of SOC decision-making so analysts spend less time on repetitive work and more time on real threats.

## Step 1: Frame the Buyer Problem

Most teams we speak with are dealing with one or more of the same issues:

- Too many alerts
- Too much manual triage
- Inconsistent severity decisions
- Slow escalation and response

## Step 2: Introduce the Flow

In this example, an alert comes in from Microsoft Sentinel. Instead of stopping with a raw alert, we pass it through an automation layer that sends the relevant context to an AI triage engine.

The AI analyzes the alert, adds reasoning, assigns severity, and recommends an action. From there, Swimlane or another orchestration layer can route or execute the next step.

## Step 3: Walk Through a Scenario

Use one scenario such as suspicious login, phishing, or endpoint isolation.

Explain:

- What triggered the alert
- What context is gathered
- What the AI determines
- What action the workflow takes next

## Step 4: Tie It To Value

The important part is not just automation for its own sake. The value is operational. We reduce repetitive analyst work, improve response consistency, and shorten time to action.

## Step 5: Close Toward Discovery

The question is not whether this can work in theory. The question is which workflow in your environment would be the best place to prove value first.

That is usually where we start: one high-volume use case, measurable outcomes, then expand.
