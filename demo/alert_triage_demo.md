# Platform-Agnostic AI Triage Demo

## Use Case: Suspicious Login Alert

### Step 1: Alert Trigger

- A suspicious login alert is generated by the client's SIEM, identity platform, or cloud security tool

### Step 2: Ingestion

- The alert is sent through an API, webhook, automation rule, or lightweight connector

### Step 3: AI Processing

Send normalized data to OpenAI:

```json
{
  "source_tool": "client_siem_or_identity_platform",
  "alert": "Multiple failed logins followed by a successful login",
  "user": "admin@company.com",
  "location": "Unknown",
  "ip": "185.220.101.44"
}
```

### Step 4: AI Output

- Classification: Suspicious
- Severity: High
- Reason: Impossible travel plus brute force pattern

### Step 5: Action

- Update the incident or ticket
- Notify the assigned analyst or team
- Trigger containment or escalation using the client's existing workflow tools
